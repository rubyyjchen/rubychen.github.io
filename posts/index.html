伊甸基金會愛心棧系統平台

計劃簡介 此系統是將伊甸基金會原本的愛心棧人工處理流程全面E化，由臺科大團隊負責從零開始建置系統平台，包含系統架構、資料庫、LINE Bot及網頁等。 計畫期間 2020 年 8 月 至今 計畫挑戰 因伊甸單位並沒有開發過類似的大型資訊系統平台，因此除了程式開發的難度以外，針對系統的架構的設計及功能呈現也須透過雙方持續溝通，我們也常經由自身的開發經驗向伊甸單位提供對於系統或使用者操作上的建議等，在種種狀況下，此專案的開發相當不易且複雜。 此專案還分為前台與後台系統，合計共三、四十種系統功能需開發，且有五種需不同功能、權限及使用平台的使用者角色，因此在這大型的系統開發初期，關於系統的建置及使用環境，也是由我們團隊經使用者需求的考量及本身的開發經驗，給予伊甸基金會建議。 負責內容 前期 系統架構設計的討論、前端網頁及LINE Bot功能開發及設計。 主要使用 HTML CSS Javascript Bootstrap進行開發 後期 代表台科大開發團隊擔任與伊甸基金會的聯絡窗口 對內: 開發團隊的進度控管與協助系統的設計與調整 對外: 與伊甸基金會窗口進行溝通聯繫 個人成果 1.建置兩個 LINE Bot 提供各自功能 2.開發約三十個網頁（前端畫面設計與撰寫） 整體成果 五個LINE Bot、一個後台系統、四十多種功能 計畫成果與活動 系統畫面 愛心棧記者會

合規新方法，QBot來幫忙

於經濟部工業局跨域數位人才加速躍升計畫(DIGI+)發表，與大同世界科技另外三名實習生共同參與 專題摘要 此專題成果應用於法遵部門向企業IT人員稽查合規之情境，結合Splunk軟體進行主機監控及異常偵測，即時使用LINE發布告警，讓Qbot機器人透過答覆流程處理異常事件 本專題的目的在設計支援腳本流程的文字機器人(QBot) 服務，有別於一般簡單交談的對話式機器人，QBot支援流程式對話的定義。需開發腳本流程的編輯工具，並產生腳本的描述檔案(JSON)，使得QBot能夠根據腳本內容，引導使用者參與流程，例如訂票、訂餐等無法透過簡單對話就完成的業務邏輯，皆可以透過腳本的引導來實現，使得用戶可以透過與機器人的會話取得服務。 負責內容 Splunk儀表板開發 成果發表簡報製作 於展場解說專題成果 作品成果
